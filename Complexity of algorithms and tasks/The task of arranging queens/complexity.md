# Анализ сложности решений задачи N-ферзей

## 1. Переборное решение (Brute force)
**Временная сложность:** O(N! * N²)
**Пространственная сложность:** O(N)

**Обоснование:**
- Генерируются все перестановки чисел от 0 до N-1 (N! вариантов)
- Для каждой перестановки проверяются N²/2 диагональных конфликтов
- Получается O(N! * N²) операций

## 2. Рекурсивное решение (Backtracking)
**Временная сложность:** O(N!)
**Пространственная сложность:** O(N)

**Обоснование:**
- Используется метод backtracking для отсечения заведомо неверных расстановок
- В худшем случае перебирается N! вариантов, но на практике значительно меньше
- Рекурсивные вызовы используют стек глубиной N

## 3. Оптимизированное решение (Bitwise backtracking)
**Временная сложность:** O(N!)
**Пространственная сложность:** O(N)

**Обоснование:**
- Используются битовые маски для быстрой проверки доступных позиций
- Каждая операция проверки выполняется за O(1) вместо O(N)
- Теоретическая сложность остается O(N!), но константа значительно уменьшена
- Практически в 5-10 раз быстрее обычного backtracking для больших N
